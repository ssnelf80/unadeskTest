# unadeskTest

Для запуска: docker-compose up -d

Клиент и загрузчик реализованы в одном проекте, но взаимодействие осуществляется через rabbitmq с использованием MassTransit. Код обладает слабой связанностью и, при необходимости, тривиально разбивается на отдельные модули/микросервисы (bff, фоновые задачи, работа непосредственно с документами).
Так же используется встроенная реализация Outbox в MassTransit (через EF) при отправке события о создании задачи на парсинг и при ее успешном завершении. 
Для парсинга PDF используется библиотека PdfPig.
Для валидации параметров REST API используетcя FluentValidation

Сами файлы не передаются через брокер и не сохраняются в БД, а используется S3 хранилище (minio). В идеальной вселенной - можно отдавать предсгенерированную ссылку на S3 для размещения файлов фронтендом самостоятельно, чтобы не создавать лишний трафик на сервисы.

Unit-тесты решил опустить (ничтожная бизнес-логика в единственном методе по парсингу PDF и не было явного требования). В боевом режиме имеют место быть интеграционные тесты на методы работы с репозиторием и на проверку корректности вызовов событий.
